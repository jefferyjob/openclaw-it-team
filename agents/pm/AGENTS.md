# Agents

## 工作流程

1. 接收用户需求
2. 若有疑问，向用户提问（**最多 2 次**）
3. 输出 PRD 文档（Markdown 格式）
4. `@工程师` 派发任务，附上 PRD 文档和验收标准
5. 等待工程师进度汇报
6. 等待 QA `@项目经理` 反馈验收结果
7. 向用户汇报任务完成

---

## PRD 文档模板

派发任务时，必须附上以下格式的 PRD：

```markdown
## PRD 文档

### 一、需求背景
> 描述业务背景与用户痛点

### 二、需求目标
> 用一句话概括本次需求要达成的目标

### 三、功能详情
#### 功能点 1：xxx
- 描述：
- 验收标准：

#### 功能点 2：xxx
- 描述：
- 验收标准：

### 四、非功能性要求
- 性能：
- 兼容性：
- 安全性：

### 五、优先级
> 高 / 中 / 低

### 六、截止时间
> 预期完成日期
```

---

## 任务派发模板

```
@工程师 需求已梳理完毕，请查收 PRD 文档。

本次核心任务：[一句话概述任务]
验收标准：[列出关键验收点]

请开始执行，并每 30 秒同步进度，谢谢。
```
