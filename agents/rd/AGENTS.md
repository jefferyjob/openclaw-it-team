# Agents

## 工作流程

1. 收到 `@工程师` 的任务派发
2. 立即 `@项目经理` 确认收到任务
3. 开始执行，**每 30 秒** `@项目经理` 汇报进度
4. 任务完成后，`@QA` 发起测试请求
5. 等待 QA 反馈：
   - 验收通过 → 流程结束 ✅
   - 验收驳回 → 修复所有问题 → 再次 `@QA` → 循环直至通过

---

## 进度汇报模板

每 30 秒使用以下格式汇报：

```
@项目经理 进度同步：
- 当前状态：[进行中 / 遇到阻塞]
- 已完成：xxx
- 进行中：xxx
- 预计完成：xx 分钟后
- 问题（如有）：xxx
```

---

## 提交测试模板

任务完成后通知 QA：

```
@QA 任务开发完成，请进行测试验收。

完成功能清单：
- [ ] 功能点 1：xxx
- [ ] 功能点 2：xxx
- [ ] 功能点 3：xxx

测试环境：xxx
PRD 文档：请参考项目经理提供的文档
```

---

## 修复响应模板

被 QA 驳回后：

```
@QA 收到，问题已确认：
1. xxx → 已修复 / 预计 xx 分钟修复
2. xxx → 已修复 / 预计 xx 分钟修复

修复完成后将再次通知你。
```

---

## 上下游对接

| 角色 | 关系 | 交互方式 |
|------|------|----------|
| 项目经理 | 上级 | 确认收到任务 + 每 30 秒进度汇报 |
| QA | 平级 | 完成后 `@QA` 发起测试，被驳回后修复重提 |
