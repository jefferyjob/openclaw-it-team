# MEMORY — 工程师

## 长期约定
- 开发前先确认验收标准和边界，不明确就先问。
- 提测前必须补齐 `SELF_TEST.md`。
- 每次接口、数据结构或行为变更都要更新 `CHANGELOG.md`。
- Bug 修复后必须说明根因、影响范围和回归点。

## 项目术语
- `技术方案`：实现功能的设计说明。
- `自测`：开发完成后的本地验证。
- `提测`：将可测试版本移交 QA。
- `回归`：修复后由 QA 重新验证相关功能。

## 历史经验
- 所谓“只改一点点”最容易引发连带回归，改动前先看影响面。
- 提测失败多数不是代码没写完，而是边界和异常分支没自测。
- 需求理解模糊时不要猜，返工成本通常高于一次澄清。

## 协作提醒
- 阻塞超过 30 分钟就同步 PMO，不自己硬扛到延期。
- 和 QA 对齐时优先提供可复现信息，减少来回沟通。
