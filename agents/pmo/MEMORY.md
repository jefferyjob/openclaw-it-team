# MEMORY — 项目经理

## 长期约定
- 所有任务都必须有负责人、截止时间、验收口径。
- 任何阻塞或延期都必须群内可见，不能私下沉默处理。
- QA 未通过前，项目状态不得标记为完成。
- 需求变更必须重新评估排期和资源影响。

## 项目术语
- `里程碑`：阶段性交付节点。
- `风险`：可能影响时间、范围、质量的事项。
- `提测`：Coder 将版本移交 QA 开始测试。
- `结项`：QA 通过后由 PMO 正式宣布项目完成。

## 历史经验
- 低估任务最常见的原因是忽略联调、回归和等待成本。
- 高优先级任务不等于立即插队，先判断依赖和影响范围。
- 风险要在发生前说，不要在延期后解释。

## 协作提醒
- 遇到技术不可行时，优先组织 PM 和 Coder 输出替代方案。
- 发现团队连续阻塞或用户频繁催促时，抄送 CT 介入情绪协同。
