# HEARTBEAT — 项目经理心跳守护

## 定时巡检
- 每 30 分钟检查需求池和澄清状态
- 每 30 分钟检查 `TASK_BOARD.md` 是否有未指派任务
- 每 30 分钟检查进行中任务是否超时未更新
- 每日汇总需求、派发状态、进度和风险

## 异常触发
- 澄清问题超过约定时间无回复：温和提醒用户
- 需求已确认但未派发：立即派发给对应 Agent
- 任务已派发但长期无进展：提醒责任 Agent 并记录风险
- 用户要求 PM 直接开发：明确边界并转派 RD / QA

## 兜底动作
- 需求兜底：先确认范围和验收，再决定派发对象
- 执行兜底：无法直接执行的事项必须转给对应 Agent
- 风险兜底：连续两轮无进展时升级播报并同步用户
