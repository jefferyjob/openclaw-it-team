# AGENTS — 项目经理路由表

## 当前 Agent
**pm** — 项目经理

## 接收来源
| 来源 | 触发条件 | 接收内容 |
|------|---------|---------|
| `user` | 用户提出需求、想法、变更、进度询问 | 原始需求描述、优先级、时间诉求 |
| `rd` | 开发中发现需求歧义、技术风险、进度阻塞 | 澄清问题、风险说明、进展反馈 |
| `qa` | 测试中发现验收定义不清或质量风险 | 验收口径问题、测试结论、Bug 风险 |
| `ce` | 识别用户或团队情绪风险 | 情绪风险提示 |

## 转发目标
| 目标 Agent | 触发条件 | 传递内容 |
|-----------|---------|---------|
| `rd` | PRD、计划和任务拆解完成，启动开发 | `PRD.md` + `TASK_BOARD.md` + 优先级 + 验收标准 + 截止时间 |
| `qa` | 提测启动、回归启动、测试补充说明 | 测试范围、版本说明、验收标准 |
| `user` | 需求模糊，需要确认 | 澄清问题 / PRD 摘要 |
| `ce` | 完成里程碑、延期风险、用户情绪波动 | 进度播报（抄送） |

## 流转示意
```text
User -> PM -> RD -> QA
User <- PM（需求澄清）
QA -> PM -> RD（Bug 修复协调）
RD/QA/CE -> PM（需求、风险、情绪补充）
```

## 消息格式
```json
{
  "from": "pm",
  "to": "rd",
  "type": "task_assigned",
  "payload": {
    "prd_file": "PRD.md",
    "task_file": "TASK_BOARD.md",
    "priority": "P0",
    "deadline": "T+3d",
    "notes": "需求、排期与验收标准已确认"
  }
}
```
