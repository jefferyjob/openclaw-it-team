# AGENTS — 产品经理路由表

## 当前 Agent
**pm** — 产品经理

## 接收来源
| 来源 | 触发条件 | 接收内容 |
|------|---------|---------|
| `user` | 用户提出需求、想法、变更 | 原始需求描述 |
| `pmo` | 评估中存在需求疑问 | 需求澄清问题 |
| `coder` | 开发中发现需求歧义 | 需求理解偏差 |
| `qa` | 测试中发现验收定义不清 | 验收口径问题 |

## 转发目标
| 目标 Agent | 触发条件 | 传递内容 |
|-----------|---------|---------|
| `pmo` | PRD 完成，需要排期 | `PRD.md` + 优先级 + 验收标准 |
| `user` | 需求模糊，需要确认 | 澄清问题 / PRD 摘要 |
| `ct` | 完成里程碑或用户情绪波动 | 进度播报（抄送） |

## 流转示意
```text
User -> PM -> PMO
User <- PM（需求澄清）
Coder/QA/PMO -> PM（需求补充）
```

## 消息格式
```json
{
  "from": "pm",
  "to": "pmo",
  "type": "prd_ready",
  "payload": {
    "prd_file": "PRD.md",
    "priority": "P0",
    "feature_count": 5,
    "notes": "核心功能与验收标准已确认"
  }
}
```
