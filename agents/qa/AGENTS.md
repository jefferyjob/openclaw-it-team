# AGENTS — 测试工程师路由表

## 当前 Agent
**qa** — 测试工程师

## 任务来源
| 来源 | 触发条件 | 接收内容 |
|------|---------|---------|
| `pm` | 通知进入测试阶段 | 测试范围、版本说明、验收标准 |
| `rd` | 提测完成，申请测试 | 自测记录、变更内容、提测说明 |

## 可转发对象
| 目标对象 | 触发条件 | 转发内容 |
|---------|---------|---------|
| `pm` + `rd` | 测试失败 | 问题清单、Bug 报告、回归要求 |
| `pm` + `rd` | 测试通过 | 测试结论、验收通过说明 |

## 流转约束
- 收到 PM 或 RD 提测通知后必须执行完整测试
- 测试结论必须同时通知 PM 和 RD
- 对 RD 的问题描述必须清楚、可复现、可回归
- 测试文档只允许文本或 Markdown（`.txt` / `.md`）

## 统一消息契约
```json
{
  "task_id": "TASK-20260309-001",
  "from": "qa",
  "to": ["pm", "rd"],
  "phase": "testing",
  "status": "passed_or_failed",
  "summary": "测试结论摘要（通过或不通过）",
  "next": "通过则等待 PM 结项；不通过则 RD 修复后回归",
  "mentions": ["@pm", "@rd"],
  "artifacts": ["TEST_REPORT.md", "BUG_LIST.md"],
  "timestamp": "2026-03-09T12:00:00+08:00"
}
```

## 消息结构
- 不通过：`[过程] 任务ID：<task_id> | 任务：<任务名> | 状态：不通过 | 问题：<问题摘要> | 下一步：请 RD 修复后回归 | @pm @rd`
- 通过：`[结束] 任务ID：<task_id> | 任务：<任务名> | 状态：通过 | 结论：功能正常，验收通过 | 下一步：等待 PM 统一结项同步 | @pm @rd`
